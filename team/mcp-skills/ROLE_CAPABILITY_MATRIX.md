# ROLE_CAPABILITY_MATRIX (1차 초안)

- 작성시각: 2026-02-22 18:xx KST
- 범위: `pm`, `tl`, `architect`, `coder_a`, `coder_b` (옵션: `main`, `mango`)
- 원칙: 최소권한, 즉시 적용 가능한 기본옵션 우선, 오남용 리스크 사전 명시

## 1) 우선순위 Top 5 (공통)
1. **요구사항/결정 로그 표준화** (pm/tl 중심)
2. **아키텍처 의사결정(ADR) + 인터페이스 계약 고정** (architect)
3. **코드 품질 게이트 자동화(테스트/린트/타입체크)** (coder_a/b)
4. **릴리즈 전 검증 체크리스트 운영** (tl + pm)
5. **권한분리(읽기/쓰기/배포 분리)와 감사로그 확보** (전 역할)

## 2) 역할별 Capability 매트릭스

> 난이도: L(낮음) / M(중간) / H(높음)

### pm
- 핵심작업
  - 백로그 우선순위화, 일정/리스크 관리, 수용기준(AC) 명확화, 이해관계자 커뮤니케이션
- MCP/Skills P1
  - 문서/이슈 추적 조회·요약 Skill
  - 일정/체크리스트 Skill
- MCP/Skills P2
  - 릴리즈 노트 자동 초안 Skill
  - KPI 요약 대시보드 조회 MCP
- MCP/Skills P3
  - 다중 프로젝트 포트폴리오 리포팅 MCP
- 기대효과
  - 의사결정 속도 향상, 우선순위 충돌 감소, 일정 예측 가능성 증대
- 오남용리스크
  - 근거 없는 우선순위 변경, 과도한 알림/과업 분배
- 최소권한
  - allow: 읽기 중심(문서/이슈/메트릭 조회), 코멘트/태깅
  - deny: 코드 직접 수정/병합, 운영 배포/시크릿 접근
- 난이도: M

### tl
- 핵심작업
  - 기술 태스크 분해, 스프린트 운영, 코드리뷰 기준 적용, 릴리즈 Go/No-Go 판단
- MCP/Skills P1
  - PR/이슈 상태 집계 Skill
  - 테스트 리포트 취합 Skill
- MCP/Skills P2
  - 병목 분석/리스크 heatmap Skill
  - 품질게이트 트렌드 MCP
- MCP/Skills P3
  - 자동 할당 최적화/로드밸런싱 MCP
- 기대효과
  - 납기 준수율 향상, 결함 조기발견, 팀 생산성 균형화
- 오남용리스크
  - 성급한 병합 승인, 지표 만능주의로 품질 왜곡
- 최소권한
  - allow: PR 리뷰/라벨/머지 승인(조건부), CI 결과 조회
  - deny: 프로덕션 직접 배포, 비인가 시크릿 열람
- 난이도: M

### architect
- 핵심작업
  - 시스템 구조 설계, 인터페이스/계약 정의, 성능·보안·확장성 기준 수립
- MCP/Skills P1
  - ADR 템플릿/레지스트리 Skill
  - 의존성/구성도 생성 Skill
- MCP/Skills P2
  - 성능/보안 점검 체크 Skill
  - 계약 테스트 템플릿 MCP
- MCP/Skills P3
  - 자동 아키텍처 영향분석 MCP
- 기대효과
  - 재작업 감소, 결합도 관리, 변경영향 예측력 향상
- 오남용리스크
  - 과설계(Over-engineering), 현실 제약 미반영 설계 강행
- 최소권한
  - allow: 설계 문서 작성/검토, Read-only 코드 탐색
  - deny: 임의 배포, 데이터 변경, 시크릿 직접 사용
- 난이도: H

### coder_a
- 핵심작업
  - 기능 구현, 단위테스트 작성, 버그수정, 코드리뷰 반영
- MCP/Skills P1
  - 로컬 테스트/린트 실행 Skill
  - 코드 검색/리팩터링 보조 Skill
- MCP/Skills P2
  - 회귀 테스트 자동화 Skill
  - 품질 리포트 생성 MCP
- MCP/Skills P3
  - 안전한 자동 수정 제안 MCP
- 기대효과
  - 개발 리드타임 단축, 회귀 결함 감소
- 오남용리스크
  - 무분별한 대량 수정, 테스트 없이 커밋
- 최소권한
  - allow: 지정 디렉토리 코드 수정, 테스트 실행
  - deny: main 직접 푸시, 운영환경 접근
- 난이도: M

### coder_b
- 핵심작업
  - coder_a와 병렬 구현, 통합 테스트 보강, 문서 동기화
- MCP/Skills P1
  - 테스트 데이터/픽스처 관리 Skill
  - 문서-코드 동기 점검 Skill
- MCP/Skills P2
  - 통합 테스트 시나리오 자동화 Skill
  - 변경영향 요약 MCP
- MCP/Skills P3
  - flaky test 분석 MCP
- 기대효과
  - 병렬 처리량 증가, 통합 이슈 조기 검출
- 오남용리스크
  - 중복구현/충돌 증가, 테스트 자원 과소비
- 최소권한
  - allow: 기능 브랜치 수정, 테스트/문서 갱신
  - deny: 보호 브랜치 강제 푸시, 승인 없는 스키마 변경
- 난이도: M

### (옵션) main
- 핵심작업: 전체 오케스트레이션, 최종 승인, 예외상황 조정
- MCP/Skills P1~P3: 전역 대시보드 → 정책 제어 → 교차팀 최적화
- 기대효과: 의사결정 일관성
- 오남용리스크: 병목/단일 실패지점
- 최소권한: allow(읽기+승인), deny(직접 구현/직접 운영변경)
- 난이도: H

### (옵션) mango
- 핵심작업: 운영 보조/개인화 자동화/알림 집계
- MCP/Skills P1~P3: 알림 집계 → 요약 자동화 → 맞춤형 추천
- 기대효과: 운영 피로도 감소
- 오남용리스크: 알림 과다, 잘못된 자동 추천
- 최소권한: allow(조회/요약), deny(배포/삭제성 작업)
- 난이도: L~M

## 3) 권한 모델 요약 (기본)
- Read/Write/Execute를 역할별 분리
- **기본 deny, 필요한 allow만 부여**
- 브랜치 보호: main direct push 금지, PR+리뷰+CI 통과 필수
- 감사로그: 주요 의사결정/머지/배포승인 이벤트 기록

## 4) 즉시 적용 기준
- 위 P1 항목부터 적용(오늘 적용 가능)
- P2는 1~2주 내 안정화
- P3는 데이터 축적 후 자동화 범위 확대
