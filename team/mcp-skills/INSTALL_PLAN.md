# INSTALL_PLAN (1차 초안, 기본옵션 즉시적용형)

## 목적
- `pm/tl/architect/coder_a/coder_b` 역할 기반 MCP/Skills를 **기본옵션으로 즉시 적용**
- 복잡한 신규 인프라 없이, 현재 저장소/CI/문서 체계에서 바로 실행

## 적용 범위
- 필수: pm, tl, architect, coder_a, coder_b
- 옵션: main, mango (2단계 이후)

## Top 5 우선순위
1. 역할별 최소권한 프로파일 확정(allow/deny)
2. P1 Skills 배치(문서/이슈/테스트/린트/리뷰 보조)
3. PR 품질게이트 강제(테스트+린트+리뷰)
4. ADR/결정로그 표준 템플릿 도입
5. 검증 체크리스트 운영 및 주간 점검

## Phase 일정 (1차 계획)

### Phase 1 — Day 0~2 (즉시 적용)
- 목표: **오늘부터 동작하는 최소세트**
- 작업
  - 역할별 권한 템플릿 생성(기본 deny)
  - P1 Skills만 활성화
  - PR 규칙(리뷰 1+, CI 통과) 강제
  - 문서 템플릿(요구사항/ADR/릴리즈노트) 배치
- 산출물
  - ROLE_CAPABILITY_MATRIX 확정본 v0.1
  - VALIDATION_CHECKLIST 초안 실행
- 성공기준
  - 모든 신규 PR이 체크리스트 통과 경로를 탑재

### Phase 2 — Day 3~7
- 목표: 팀 운영 안정화
- 작업
  - P2 Skills 점진 활성화(트렌드/리포팅/통합테스트 보강)
  - 실패 케이스(권한거부/품질게이트 실패) 대응 루프 정착
  - 주간 리스크 리뷰(오남용 징후 점검)
- 산출물
  - RISK_ROLLBACK v0.2 업데이트
  - 역할별 KPI 베이스라인
- 성공기준
  - PR 리드타임/실패율 지표의 초기 안정화 확인

### Phase 3 — Week 2~4
- 목표: 고도화 자동화(P3 파일럿)
- 작업
  - 자동 영향분석/회귀탐지 등 P3 파일럿
  - 옵션 역할(main/mango) 제한적 도입
  - 권한·감사 정책 리파인
- 산출물
  - 운영 가이드 v1.0
  - P3 사용범위/중단조건 명시
- 성공기준
  - 품질/속도 지표 개선 + 리스크 허용범위 내 유지

## 즉시 적용 가능한 기본옵션 목록 (설치형)
- 문서 템플릿: Markdown 파일 추가/사용
- 코드 품질게이트: 기존 CI에 테스트/린트/타입체크 활성
- 브랜치 보호룰: main direct push 차단, PR 필수
- 역할별 실행 규칙: read/write 범위 및 금지행위 명문화
- 리포트 자동화: 기존 로그/테스트 결과 요약 스크립트 사용

## 비포함(이번 1차에서 제외)
- 신규 외부 SaaS 도입
- 대규모 인프라 변경
- 운영 배포 자동승인

## 의존성
- 저장소 권한 설정 가능해야 함
- CI 파이프라인 수정 권한 필요
- 팀 합의된 역할 정의 필요

## 리스크 완화
- 점진적 활성화(Phase별)
- 실패 시 즉시 롤백(권한/자동화 범위 축소)
- 감사로그 기반 사후분석
