# 운영 체크리스트 (1페이지)

목적: **실계좌(업비트/빗썸) 단일 계좌** 위에서 전략을 안전하게 운용하고, 일일 성과/리스크를 빠르게 점검하기 위한 운영표준.

---

## 1) 일일 점검 루틴 (D-Start / Intraday / D-End)

### D-Start (장 시작 전 5~10분)
- [ ] 거래소 연결 상태/API 키 상태 정상
- [ ] 전략 활성화 상태, 심볼/주문 파라미터 확인
- [ ] 금일 `capital_cap_krw` 확인 (전일 대비 변경 여부)
- [ ] 리스크 한도 로드 확인 (`daily_loss_limit`, `max_mdd_limit`, `max_trades_per_day`)
- [ ] 알림 채널(텔레그램/슬랙) 헬스체크

### Intraday (운영 중, 1~2시간 간격)
- [ ] 실현/미실현 PnL 점검 (가상 원장 기준)
- [ ] 현재 MDD 점검 (당일/누적)
- [ ] 거래 건수 점검 (`trades_today`)
- [ ] 슬리피지/수수료 급증 여부 점검
- [ ] 이상징후(미체결 누적, 재시도 폭증, 레이턴시) 점검

### D-End (마감 후 10분)
- [ ] 일일 PnL 확정 기록
- [ ] 당일 최대 MDD 기록
- [ ] 거래 건수/승률/평균손익 기록
- [ ] 차단 이벤트(거래중지, 손실한도 초과) 발생 여부 기록
- [ ] 다음날 `capital_cap_krw` 유지/조정 결정

---

## 2) 핵심 KPI / 임계치

| 지표 | 정의 | 정상 범위(예시) | 경고 | 중지(킬스위치) |
|---|---|---:|---:|---:|
| 일일 PnL | 당일 확정 + 평가손익(가상 원장) | >= -1.5% | <= -2.0% | <= -3.0% |
| 누적 MDD | 피크 대비 하락률 | <= 6% | >= 8% | >= 10% |
| 일일 거래건수 | 체결 기준 주문 라운드 | 5~60건 | >80건 | >120건 |
| 주문 실패율 | 실패/전체 주문 | <2% | >=3% | >=5% |
| 평균 슬리피지 | 체결가-기준가 절대편차 | <0.15% | >=0.25% | >=0.40% |

> 임계치는 전략별 변동성에 맞춰 별도 프로파일로 관리.

---

## 3) 알림 기준 (Severity)

### INFO
- 목표 범위 내 운영
- 일일 리포트(마감 요약)

### WARN
- 일일 PnL -2% 이하
- 거래건수 경고 초과
- 주문 실패율 3% 이상

### CRITICAL
- 일일 PnL -3% 이하 (즉시 신규 진입 중단)
- 누적 MDD 10% 이상 (전략 자동 중지)
- API 장애 지속(예: 3분+) 또는 잔고 동기화 실패

---

## 4) 운영 의사결정 규칙 (간단 버전)

- **Rule-1 (손실제한):** `daily_pnl_pct <= -3.0%` 이면 자동 `HALT_NEW_ENTRY`.
- **Rule-2 (MDD):** `mdd_pct >= 10%` 이면 `STRATEGY_STOP` + 수동승인 전 재개 금지.
- **Rule-3 (과매매):** `trades_today > max_trades_per_day` 이면 당일 추가진입 금지.
- **Rule-4 (기술장애):** 주문 실패율 급증/거래소 장애 시 즉시 보호모드 전환(청산 우선).
